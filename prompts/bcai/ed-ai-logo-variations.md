# ED + AI Logo Variations — AI Education Meetup

Brand extension for the **BC + AI Ecosystem**: a standalone identity for the
**ED + AI** education meetups that locks up cleanly with the parent BC + AI mark.

**Base Identity:**
- Parent brand: BC + AI Ecosystem (the `[X] + AI` family with the signature plus glyph)
- Sub-brand: **ED + AI** — the AI Education Meetup
- Naming logic: ED + AI sits in the same family as BC + AI, CV + AI, etc. The
  shared element is the **"+"** — same glyph, same accent color, same energy.
- Positioning: welcoming, practical, educator-led, community-rooted (not edtech sales)
- Audience: teachers, instructors, faculty, instructional designers, ed-curious builders

**Design Goals:**
- A confident **ED + AI** wordmark that can stand on its own
- A descriptor line reading **"AI Education Meetup"** underneath
- A clear **lockup with the BC + AI logo** so the parentage reads instantly
- Reuse the BC + AI plus glyph and palette so the family is obvious
- Works at avatar size and on an event banner

**Output root:** `output/ed-ai-logo/`

**Reference image:** Pass the official BC + AI logo as a reference so every
variation inherits the real plus glyph, letterforms, and exact palette rather
than inventing them. Match the reference logo's colors and the treatment of its
"+". Do not invent a new color system.

**Text rule (logo work — opposite of the communities graphics kit):** Baked-in
text **is wanted** here. The wordmark must read **ED + AI**, the descriptor
**AI Education Meetup**, and endorsement lockups may read **a BC + AI community**
or **part of the BC + AI ecosystem**. Keep all type crisp, correctly spelled,
and evenly spaced — no garbled generated lettering.

**Shared negative constraints:** no garbled or misspelled text, no fake/extra
logos beyond BC + AI and ED + AI, no generic AI clip art, no robot heads or
robot hands, no glowing brain circuits, no neon circuit-board cliché, no SaaS
gradient blobs, no stock conference imagery, no crypto/web3 vibes, no
chalkboard slogans, no children as the subject.

## Design Notes

**Family logic:**
- ED + AI is one of the `[X] + AI` BC + AI sub-brands. The constant is the "+".
- Always reuse the reference logo's plus glyph, accent color, and palette.
- Pull exact colors from the reference image — do not invent a new system.

**Typography:**
- Clean modern sans-serif, bold for "ED + AI", medium/light for the descriptor.
- "AI Education Meetup" sits directly under the wordmark, smaller, well-spaced.
- All caps reserved for seals and endorsement micro-copy.

**ED + AI identity:**
- Welcoming, practical, educator-led; community warmth over edtech sales gloss.
- The "+" stays the literal BC + AI plus glyph in every variation.
- Education cues (books, lesson cards, learning network) appear only in the
  surrounding composition of variant #7 — never by altering the mark, never as
  generic clip art.

**Generation command (pass the BC + AI logo as the reference):**

Use `--reference-role brand` so the BC + AI mark is *preserved* in the lockup
rather than just borrowed for look-and-feel, and `--global-reference-images` so
the same logo is reused across every variation:

```bash
python generate.py -f prompts/bcai/ed-ai-logo-variations.md -d output/ed-ai-logo \
  --style bcai-ecosystem -m gemini-3-pro-image-preview \
  --global-reference-images "/absolute/path/to/bc-ai-logo.png" \
  --reference-role brand -w 2
```

Or a single variation via the CLI:
```bash
npx rafiki --prompt "<paste one prompt above>" \
  --style bcai-ecosystem \
  --reference-image "/absolute/path/to/bc-ai-logo.png" \
  --reference-role brand \
  --aspect-ratio 1:1 \
  --output ed-ai-primary.png
```

Review with the local portal, then export approved marks:
```bash
python generate.py serve --open
```
